# Instructions permanentes

## Regle absolue
Ne PAS modifier SPECIFICATION.md sans instruction
explicite. Ce fichier est le contrat de reference,
écrit et valide en amont. Si une contradiction
apparait entre la specification et une demande qui
ne mentionne pas explicitement SPECIFICATION.md,
SIGNALE-LA et attends une decision. Ne tranche pas
seul.

Une demande qui cible nommement SPECIFICATION.md
et decrit precisement les modifications a apporter
vaut instruction explicite : l'appliquer directement,
sans redemander confirmation.

## Contenu pedagogique
N'invente JAMAIS de contenu pedagogique.
Toutes les notions doivent provenir de l'analyse
du livret ministeriel, en cours.
Si un fichier de donnees est vide, laisse-le vide.

## Contraintes techniques
- HTML, CSS, JavaScript natifs uniquement
- Aucun framework, aucun build, aucune dependance
- Doit fonctionner sur GitHub Pages
- Mobile d'abord
- Respecter la checklist de conformite (section 10)

## Methode
- Une tache = une fonctionnalite
- Lire SPECIFICATION.md avant chaque tache
- Ne pas anticiper les etapes suivantes
